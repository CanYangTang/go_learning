## 学习前问题

- 什么是裸 return？

## 答疑记录

- 裸 return 指函数有具名返回值时，`return` 后面不写具体返回内容，直接返回当前具名返回变量的值。
- 裸 return 依赖具名返回值；如果函数没有具名返回值，就不能用裸 return 返回结果。
- 具名返回值在函数进入时也有零值，例如 `float64` 是 `0`，`bool` 是 `false`。
- 学习阶段和多数业务代码中，优先使用显式返回，例如 `return a / b, true`，可读性更好。

## 今日小测试

1. Go 的多返回值有什么用？今天的 `Divide` 为什么要返回 `(float64, bool)`？
   - 回答：显式处理错误/返回多个结果；除法中的除数不能为 0。
   - 结果：通过。
   - 标准答案：多返回值可以让函数一次返回主要结果和附加状态。`Divide` 返回 `(float64, bool)`，第一个值表示计算结果，第二个值表示除法是否成功；当除数为 0 时返回 `0, false`。
2. 变参函数 `func Sum(nums ...float64)` 中，`nums` 在函数内部可以当成什么类型使用？
   - 回答：slice。
   - 结果：通过。
   - 标准答案：`nums` 在函数内部可以当成 `[]float64` 使用，因此可以用 `for range` 遍历。
3. `Apply(a, b float64, op func(float64, float64) float64)` 里，`op` 是什么？为什么 `Apply(2, 3, Add)` 可以工作？
   - 回答：`op` 是一个函数名；因为内部函数可以访问外部函数的变量，形成一个闭包。
   - 结果：需要修正。
   - 标准答案：`op` 是一个函数类型的参数，不是函数名。`Apply(2, 3, Add)` 可以工作，是因为 `Add` 的函数签名正好是 `func(float64, float64) float64`，能作为参数传给 `op`。这里不涉及闭包。
4. 什么是匿名函数？今天测试里的匿名函数做了什么？
   - 回答：匿名函数就是没有名字的函数；将匿名函数定义后立即执行。
   - 结果：基本通过，但需要修正今天测试里的场景。
   - 标准答案：匿名函数是没有名字的函数。今天测试里的匿名函数作为参数传给 `Apply`，逻辑是 `return a*b + 1`；它不是定义后立刻直接执行，而是在 `Apply` 内部通过 `op(a, b)` 被调用。
5. 什么是闭包？为什么 `NewCounter()` 返回的函数能记住之前的 `count`？
   - 回答：闭包是一个函数捕获了外部变量；因为 `NewCounter()` 形成了一个闭包。
   - 结果：通过。
   - 标准答案：闭包是函数捕获并持续引用外部作用域变量。`NewCounter()` 内部定义了 `count`，返回的匿名函数捕获了这个 `count`，所以每次调用同一个返回函数时都会继续修改同一个 `count`。

## 测试结果

- 3 题通过，2 题需要修正或补充。
- 需要重点巩固：函数作为参数和闭包是两个不同概念；匿名函数可以作为参数传递，不一定是立即执行。

## 遇到的问题

- 裸 return 容易降低可读性，学习和业务代码中优先使用显式返回。
- 容易把“函数作为参数”和“闭包”混在一起；`Apply(2, 3, Add)` 是函数作为参数，不是闭包。
- 容易把匿名函数都理解成立即执行；匿名函数可以立即执行，也可以赋值给变量或作为参数传递。

## 关键收获

1. 函数可以通过参数接收输入，并通过返回值输出结果；纯函数比直接打印更容易测试。
2. Go 支持多返回值，常用于同时返回结果和状态，后续错误处理中会扩展为 `result, err` 模式。
3. 变参函数在函数内部可以像 slice 一样使用，并可通过 `for range` 遍历。
4. 函数可以作为参数传入，前提是被传入函数的签名和参数类型一致；`Apply(2, 3, Add)` 是函数作为参数，不是闭包。
5. 匿名函数没有名字，可以赋值给变量、作为参数传递，或定义后立即执行。
6. 闭包可以捕获外部变量，并在多次调用之间保存状态；`NewCounter()` 返回的匿名函数捕获了同一个 `count`。
